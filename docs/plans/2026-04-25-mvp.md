# Knowledge Brain MVP — Implementation Plan

**Date:** 2026-04-25
**Status:** Approved by maintainer; in execution

**Goal:** Smallest possible brain that stores knowledge nodes in SQLite, exposes
`brain_write` / `brain_query` MCP tools, ships a `brain` CLI, returns Pydantic
models (deterministic JSON), and enforces input-token limits.

**Out of scope (do NOT build without explicit approval):** embeddings, semantic
search, graph edges, the 5 knowledge types, LLM Compiler, ranking, citations,
self-maintenance, plugins, ambient retrieval, team sync, observability, health
checks. See `CLAUDE.md`.

---

## Locked design decisions

- Tag query semantics: AND across all supplied tags
- Combined query+tags: tag-filter first, then text-match content within (`AND`)
- Token counter: word-based, `round(len(text.split()) * 1.3)`
- MCP integration test: call registered tool functions directly + a thin test
  that asserts the server registers `brain_write` and `brain_query`. No
  subprocess (Windows reliability).
- Tag SQL: `EXISTS (SELECT 1 FROM json_each(tags) WHERE value = ?)` per tag, ANDed
- Text SQL: `LOWER(content) LIKE LOWER(?) ESCAPE '\'` with `%`/`_`/`\` escaped
- DB path: global `--db-path` flag on every CLI command; default
  `data_store/knowledge.db`
- `brain init`: refuses if DB exists; `--force` recreates
- Schema CHECK: `confidence BETWEEN 0.0 AND 1.0` + `source_type` enum check
- CLI default `source_type=human`; MCP default `source_type=session`

## Tasks

### Task 0 — Scaffolding (this commit)

Files: `pyproject.toml`, `.gitignore`, `README.md`, `CLAUDE.md`, package
`__init__.py` files, `tests/conftest.py` with `tmp_db_path` fixture, this plan.

Verify: `uv sync --extra dev` succeeds; `uv run pytest` collects 0 tests.

### Task 1 — `models.py`

`KnowledgeNode`, `WriteResult`, `CompiledContextResponse`, `InputValidationError`.

Tests:
- Round-trip via `model_dump_json` / `model_validate_json`
- Default `id` matches `kn-` + 12 hex chars
- Default `created_at` is ISO 8601 UTC parseable by `datetime.fromisoformat`
- `confidence` out of `[0,1]` raises `ValidationError`
- `source_type` not in `{session,human,ingestion}` raises
- JSON key order matches declaration order
- `InputValidationError` is a `ValueError` subclass

### Task 2 — `token_utils.py`

`count_tokens(text)` returns `round(len(text.split()) * 1.3)`.

Tests: empty → 0; "hello" → 1; "hello world" → 3; whitespace-only → 0.

### Task 3 — `quality.py`

`validate_write_input(content, tags, max_input_tokens)` and
`validate_query_input(query, max_input_tokens)`.

Tests: accept happy path; reject empty/whitespace content; reject empty tags;
reject content over token limit (with token count in message); same for query.

### Task 4 — `store.py`

`Store(db_path)` with `init_schema`, `write_node`, `query`.

Tests: schema creation idempotent; round-trip write/query; case-insensitive
LIKE; LIKE escape (literal `%` / `_` doesn't act as wildcard); tag AND match;
exact-tag match (no substring matches like `tag-foo` matching `foo`); combined
query+tags; ordered by `created_at DESC`; `limit` caps results, `total_matches`
is unbounded count; `CHECK` rejects out-of-range confidence and bogus
`source_type`.

### Task 5 — CLI

`cli/main.py` (argparse, global `--db-path`, subparsers), `cli/init.py`,
`cli/write.py`, `cli/query.py`, `cli/list.py`. CLI default `source_type=human`.

Integration tests via direct `main([...])` calls:
- init (and re-init refusal/`--force`)
- write happy path; reject bad input via exit 1
- query (text only, with tags, combined)
- list with limit
- All output is valid JSON for `query`/`list`

### Task 6 — MCP server

`mcp/server.py` instantiates `FastMCP("knowledge-brain")`, registers
`brain_write` and `brain_query`. `main()` runs over stdio. DB path from
`BRAIN_DB_PATH` env, default `data_store/knowledge.db`.

Integration tests:
- Server registers both tools (introspect FastMCP registry)
- Direct tool-function calls round-trip a node
- Empty content raises `InputValidationError`
- Uses temp DB via monkeypatched env

## Final verification

`uv run pytest -v` green; `uv run ruff check .` clean; `uv run ruff format
